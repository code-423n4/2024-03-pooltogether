How does the yield buffer work? It's kinda confusing
RaymondFam 03/06/2024 00:58
Under normal operation, (totalAssets >= totalDebt) is mandated to facilitate deposit. For example, if you don't have yield buffer in place for a newly and permissionlessly deployed Prize Vault, after the first deposit is made it will turned into lossy state right away because the yield vault would round down each time a deposit/withdraw is made by a single unit (or 1 wei if 18 decimal token is entailed). In the NatSpec comment of yieldBuffer, an example where a reasonable yield buffer for USDC with 6 decimals might be 1e5 ($0.10), which will cover up to 100k rounding errors while still being an insignificant value is given. It simply means the yield vault could afford to do nothing while facilitating 100k deposits/withdraws. (edited)
